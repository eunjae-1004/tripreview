## 📋 다음 단계

### 1. GitHub 저장소 생성 및 연결

1. **GitHub에서 저장소 생성**
   - https://github.com/new 접속
   - Repository name: `tripreview` (또는 원하는 이름)
   - Public 또는 Private 선택
   - "Create repository" 클릭

2. **로컬 저장소와 연결**
   ```bash
   git remote add origin https://github.com/your-username/tripreview.git
   git branch -M main
   git push -u origin main
   ```

### 2. Railway 배포 (Server)

#### 2.1 프로젝트 생성
1. https://railway.app 접속
2. "New Project" 클릭
3. "Deploy from GitHub repo" 선택
4. 방금 생성한 `tripreview` 저장소 선택
5. "Deploy Now" 클릭

#### 2.2 Root Directory 설정
1. 프로젝트 대시보드에서 "Settings" 탭 클릭
2. "Root Directory" 섹션 찾기
3. `server` 입력 후 저장

#### 2.3 PostgreSQL 데이터베이스 추가
1. 프로젝트에서 "New" 버튼 클릭
2. "Database" → "Add PostgreSQL" 선택
3. 자동으로 `DATABASE_URL` 환경 변수가 생성됨

#### 2.4 환경 변수 설정
프로젝트 → "Variables" 탭에서 다음 추가:

```
ADMIN_PASSWORD=your-secure-password-123
JWT_SECRET=generate-random-string-here
NODE_ENV=production
PORT=3000
```

**JWT_SECRET 생성 방법:**
```bash
# Windows PowerShell
[Convert]::ToBase64String([System.Text.Encoding]::UTF8.GetBytes([System.Guid]::NewGuid().ToString() + [System.Guid]::NewGuid().ToString()))
```

#### 2.5 데이터베이스 마이그레이션
Railway CLI 사용:
```bash
npm i -g @railway/cli
railway login
railway link
cd server
railway run npm run migrate
```

또는 Railway PostgreSQL 콘솔에서:
1. PostgreSQL 서비스 클릭
2. "Data" → "Query" 탭
3. `server/src/db/schema.sql` 파일 내용 복사하여 실행

#### 2.6 서버 URL 확인
1. Settings → Networking
2. 생성된 URL 복사 (예: `https://tripreview-production.up.railway.app`)
3. `/health` 엔드포인트로 테스트

### 3. Vercel 배포 (Client)

#### 3.1 프로젝트 생성
1. https://vercel.com 접속
2. "Add New..." → "Project" 클릭
3. GitHub 저장소에서 `tripreview` 선택
4. "Import" 클릭

#### 3.2 Root Directory 설정
1. "Configure Project" 화면
2. "Root Directory" 섹션에서 "Edit" 클릭
3. `client` 입력
4. "Continue" 클릭

#### 3.3 환경 변수 설정
"Environment Variables" 섹션에서 추가:

```
NEXT_PUBLIC_API_URL=https://your-railway-app.railway.app
NEXT_PUBLIC_ADMIN_SECRET=your-secure-password-123
```

**중요:** `NEXT_PUBLIC_ADMIN_SECRET`은 Railway의 `ADMIN_PASSWORD`와 **동일**하게 설정해야 합니다.

#### 3.4 배포 실행
1. "Deploy" 버튼 클릭
2. 배포 완료 대기 (약 2-3분)
3. 생성된 URL로 접속하여 확인

### 4. 배포 확인

#### Server 확인
브라우저에서 접속:
```
https://your-railway-app.railway.app/health
```
예상 응답: `{"status":"ok"}`

#### Client 확인
1. Vercel에서 생성된 URL 접속
2. 관리자 페이지 로드 확인
3. "작업 시작" 버튼 테스트
