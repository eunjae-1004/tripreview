# 네이버맵 리뷰 10건만 수집되는 현상 검토

## 결론: **10건으로 제한하는 코드는 없음**

코드 전체를 검색한 결과, 네이버맵 전용으로 “리뷰 10건까지만 수집”하거나 “10개에서 break”하는 **명시적 제한은 없습니다**.

---

## 왜 10건에서 끝나는가

1. **첫 페이지에 10개만 DOM에 있음**  
   네이버맵 리뷰는 첫 로드 시 보통 **10개**만 렌더링됩니다.

2. **“더보기”를 못 찾거나 클릭 실패**  
   첫 10건 처리 후 **더보기 버튼**을 찾아 클릭해야 다음 10건이 로드되는데,  
   - 버튼 선택자가 현재 UI와 맞지 않거나  
   - 버튼이 뷰 밖에 있어서 보이지 않거나  
   - 클릭이 실패하면  
   → 더보기를 누르지 못하고 **그대로 루프를 빠져나와** 10건만 수집된 것처럼 보입니다.

3. **흐름 요약**  
   - `currentPageReviewCount` = DOM 리뷰 개수 (첫 페이지 ≈ 10)  
   - 0 ~ (currentPageReviewCount-1) 까지 처리 → 10건 수집  
   - 더보기 버튼 탐색 → **못 찾거나 클릭 실패** → `break`  
   - 결과: **모든 기업이 10건씩만 저장**

---

## 관련 코드 위치 (제한 없음, 참고용)

| 위치 | 내용 |
|------|------|
| **624~782** | locator 경로: `while (!shouldStop)` → 현재 페이지 개수만큼 처리 → 더보기 여러 선택자 시도 → 실패 시 `break` |
| **735~779** | 더보기 선택자 목록 + 15건 이하일 때 스크롤 후 1회 재시도 |
| **789~886** | evaluate 경로 (locator로 0건일 때만 사용) |
| **919~928** | `(i + 1) % 10 === 0` → **로그만 10개마다 출력**, 수집 개수 제한 아님 |
| **1799~1875** | 두 번째 while 경로의 더보기: locator 경로와 동일한 `moreButtonSelectorsSecond` + getByText + evaluate fallback → 없거나 실패 시 `break` |

`slice(0, 10)`, `Math.min(..., 10)`(리뷰 수 제한), “최대 10건” 같은 **리뷰 개수 제한 로직은 없습니다**.  
(3125라인 `Math.min(count, 10)`은 **아고다 호텔 링크 매칭**용이며, 네이버맵과 무관합니다.)

## 권장 대응

1. **더보기 버튼 선택자/노출 보강**  
   - iframe 안 **리뷰 리스트 컨테이너**(예: `ul#_review_list` 부모)를 **스크롤**해서 더보기 버튼이 보이게 한 뒤,  
   - 텍스트/역할 기반 탐색 추가: `getByText('더보기')`, `getByRole('button', { name: /더보기/ })` 등.

2. **실제 동작 확인**  
   - 서버/터미널 로그에서  
     `[네이버맵] 더보기 클릭 성공` vs  
     `[네이버맵] 더보기 버튼 없음 또는 클릭 불가 → 수집 종료. 누적 N개`  
     가 어떻게 찍히는지 확인하면, “10건에서 멈추는” 이유가 더보기 실패인지 바로 구분할 수 있습니다.

3. **UI 변경 대응**  
   - 네이버맵이 “더보기” 문구나 DOM 구조를 바꾼 경우,  
     **개발자 도구로 실제 페이지의 더보기 요소(태그/클래스/텍스트)를 재확인**한 뒤  
     `scraper.js` 내 `moreButtonSelectors` / `moreButtonSelectorsSecond` 및 evaluate fallback 로직에 반영하는 것이 좋습니다.  
   - 리뷰 목록 더보기(다음 페이지)는 **리스트 하단 버튼만** 클릭하도록, `ul#_review_list`의 `li` 밖에 있는 "더보기"를 클릭하는 evaluate fallback이 추가되어 있습니다.
